# IRCTC Feature Specifications — Part B

## Feature 1: Tatkal Smart Queue System

### Problem

Tatkal bookings fail during traffic spikes.

### Proposed Solution

Introduce a virtual queue system before booking.

### Key Features

* Queue position display
* Estimated waiting time
* Auto-refresh queue status
* Booking slot reservation

### Success Metric

Reduce Tatkal booking failures by 40%.

---

## Feature 2: Persistent Search Filters

### Problem

Search filters reset unexpectedly.

### Proposed Solution

Save filter selections throughout the session.

### Key Features

* Persistent filters
* Filter summary bar
* One-click reset

### Success Metric

Reduce repeated filter applications by 50%.

---

## Feature 3: Reliable Seat Preference Tracking

### Problem

Berth preferences disappear during booking.

### Proposed Solution

Track and display selected preference across all booking stages.

### Key Features

* Seat preference confirmation
* Visual seat summary
* Preference lock indicator

### Success Metric

Reduce seat-related booking complaints.

---

## Feature 4: Session Expiry Warning

### Problem

Users lose progress due to timeout.

### Proposed Solution

Show warning before session expiration.

### Key Features

* Countdown timer
* Extend session button
* Auto-save progress

### Success Metric

Reduce booking abandonment.

---

## Feature 5: PNR/TDR Guidance Layer

### Problem

Users struggle with railway terminology.

### Proposed Solution

Add contextual help and explanations.

### Key Features

* Tooltips
* Glossary
* Guided help panel

### Success Metric

Reduce support-related queries.

---

## Feature 6: Mobile Booking Optimization

### Problem

Excessive scrolling on mobile.

### Proposed Solution

Convert booking into a multi-step mobile wizard.

### Key Features

* Step-by-step forms
* Sticky action buttons
* Progress indicator

### Success Metric

Improve mobile booking completion rate.
